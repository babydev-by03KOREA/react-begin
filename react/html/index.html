JS와 REACT의 차이

`+1` 버튼이 눌린다면, id가 number인 DOM을 선택해서 innerText 속성을 1씩 더해줘라.
만약 인터랙션이 자주 발생하고, UI가 동적으로 변한다면, 이러한 규칙이 다양해지고 관리가 힘들어진다.
그렇기 때문에 JS 특정 값이 바뀌면 특정 DOM의 속성이 바뀌도록 연결을 해줘서, 업데이트 간소화를 해주는 Ember, AngularJS등이 나왔습니다.
그러나 REACT는 `DOM을 어떻게 업데이트 하는가`가 아닌 `다 날리고 모든걸 새로 보여주는건 어떨까?`라는 아이디어에서 비롯되었습니다.
그렇지만 동적인 UI를 보여주기 위해선 모든걸 다 날리고 새롭게 만들어야 한다면 속도가 굉장히 느릴겁니다. 하지만 REACT에서는 Virtual DOM을 사용해 이를 가능케 하였습니다.
